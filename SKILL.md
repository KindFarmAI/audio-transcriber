# Audio Transcriber

## Metadata
- **name:** audio-transcriber
- **description:** Транскрибация длинных аудиофайлов любой длины с обходом лимитов ASR API (30 секунд). Автоматически конвертирует, режет на чанки, транскрибирует и склеивает результат. Опционально переводит на любой язык.
- **version:** 1.0.0
- **author:** KindFarmAI
- **tags:** audio, transcription, asr, translation, speech-to-text
- **tools:** ffmpeg, asr-cli, llm

## Triggers
Используй этот скилл когда пользователь:
- Просит транскрибировать аудиофайл (mp3, wav, m4a, ogg, flac)
- Просит перевести аудио/песню/подкаст
- Говорит "распознай речь", "переведи аудио", "что говорит в файле"
- Прикрепляет аудиофайл и просит его обработать

## Instructions

### Шаг 1: Определить реальную длину аудио

ВАЖНО: Нельзя доверять метаданным файла (ffprobe format duration может показывать неверное значение). Всегда проверяйте реальную длину через декодирование.

```bash
# Правильный способ узнать реальную длину:
ffmpeg -i "INPUT_FILE" -f null - 2>&1 | grep "time="
```

Или через oneliner:
```bash
ffprobe -v error -show_entries format=duration -of default=noprint_wrappers=1:nokey=1 "INPUT_FILE"
```

Если длина > 30 секунд — нужно резать на чанки.

### Шаг 2: Конвертировать в WAV 16kHz моно

ASR API поддерживает только WAV и WebM. Любой формат нужно конвертировать:

```bash
ffmpeg -i "INPUT_FILE" -ar 16000 -ac 1 "OUTPUT.wav" -y
```

### Шаг 3: Нарезать на чанки по 29 секунд

29 секунд — безопасный размер (лимит API 30 секунд, оставляем запас):

```bash
ffmpeg -i "OUTPUT.wav" -f segment -segment_time 29 "chunk_%03d.wav" -y
```

### Шаг 4: Транскрибировать каждый чанк

```bash
for i in $(seq 0 N); do
  z-ai asr -f "chunk_$(printf '%03d' $i).wav" -o "result_$(printf '%03d' $i).json"
done
```

Затем извлечь текст из каждого JSON:
```python
import json
for i in range(N+1):
    with open(f'result_{i:03d}.json') as f:
        data = json.load(f)
    print(data.get('text', ''))
```

### Шаг 5: Объединить результаты

Склеить весь текст в один документ, убрать дубликаты на стыках чанков (последние 5 слов предыдущего чанка могут совпадать с первыми 5 словами следующего).

### Шаг 6: (Опционально) Перевести

Если пользователь просит перевод — передать объединённый текст в LLM с инструкцией перевести на нужный язык.

## Workflow

```
Пользователь даёт аудиофайл
        |
        v
Проверить реальную длину (ffmpeg decode)
        |
        +--- Если <= 29 сек ---> Транскрибировать напрямую
        |
        +--- Если > 29 сек -----> Конвертировать в WAV
                                    |
                                    v
                                Нарезать на чанки (29 сек)
                                    |
                                    v
                                Транскрибировать каждый чанк
                                    |
                                    v
                                Объединить текст
                                    |
                                    v
                          Нужен перевод? --да--> Перевести через LLM
                                    |
                                   нет
                                    v
                            Вернуть результат
```

## Output Format

Возвращай результат в таком формате:

```
## Транскрипция: [имя файла]

**Длительность:** X:XX
**Формат:** оригинальный формат файла
**Язык:** определённый язык (если возможно)

### Текст:

[объединённый текст транскрипции]

---
### Перевод (если запрошен):

[переведённый текст]
```

## Важные правила

1. ВСЕГДА проверяй реальную длину аудио через декодирование, а не через метаданные
2. Максимальный размер чанка — 29 секунд (не 30, с запасом)
3. Если текст песни — проверь наличие встроенных lyrics (metadata) в файле
4. Псевдоязыки (псевдолатынь, выдуманные языки) — предупреди пользователя что точный перевод невозможен
5. После транскрипции — удаляй временные файлы (чанки)
6. Если чанк не транскрибируется (пустой/инструментальный) — пропусти его
7. Для качественной транскрипции используй 16kHz sample rate
